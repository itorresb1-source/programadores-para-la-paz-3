/Reflexión:-[¿Por qué es importante que los proyectos de software mantengan un registro de cambios usando herramientas como Git?] 
*Respuesta: -[Sin herramientas como Git que facilitan el registro de cambios mediante copias virtuales, probablemente el proceso de programación estaría lleno de errores graves difíciles de identificar en códigos masivos y compilados, producto de que no se tendría un historial detallado de las modificaciones al código fuente, si el sistema se rompe un día, con Git se puede ver que linea de código se cambio, quien lo hizo y porque, permitiendo regresar en un tiempo corto a una versión estable anterior en cuestión de segundos, minimizando el tiempo de inactividad de una aplicación en producción. Básicamente, funciona como un "botón de deshacer" definitivo, adicionalmente permite que múltiples programadores trabajen en el mismo proyecto simultáneamente mediante ramas, evitando que uno sobrescriba el código del otro. Por otro lado, deja una bitácora exacta de qué se cambió, quién lo hizo y por qué, facilitando la detección de errores y la organización del equipo. Y finalmente, se pueden probar nuevas ideas o funciones en un entorno aislado sin arriesgar ni alterar el código que ya funciona de otra persona]
